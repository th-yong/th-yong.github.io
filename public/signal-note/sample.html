Why isn’t LLM reasoning done in vector space instead of natural language? 어제
LLMReasoningResearch
TL;DR. LLM 추론을 자연어 대신 잠재·벡터 공간에서 수행하지 않는 이유를 묻는 논의
  • 체인 오브 소트(chain-of-thought)처럼 드러나는 자연어 추론과 내부 고차원 벡터 연산의 관계에 대한 문제 제기
  • 중간 추론을 텍스트로 쓰지 않고 잠재 공간(latent space)에서 직접 전개할 수 있는지에 대한 탐색
  • 벡터 기반 추론이 더 효율적이거나 강력할지, 반대로 해석 가능성과 제어 가능성을 잃을지에 대한 토론 주제
  • LLM이 언어 생성 모델인 만큼 학습 목표와 출력 형식이 자연어 중심으로 설계된 점을 재검토하는 관점
왜 중요한가 최근 LLM 추론은 자연어 체인 오브 소트에 크게 의존해 보이지만, 실제 계산은 이미 벡터 공간에서 이뤄진다. 이 질문은 추론 표현과 실제 연산을 분리해 더 효율적이고 통제 가능한 추론 구조를 설계할 수 있는지 점검하게 한다.
배경 지식 트랜스포머는 토큰을 임베딩 벡터로 변환해 연산하고, 최종적으로만 자연어 토큰을 출력한다. 체인 오브 소트는 모델의 내부 계산 전체가 아니라, 추론 과정을 텍스트로 외부화한 표현에 가깝다.
추천 대상 LLM 추론 메커니즘, 잠재 추론(latent reasoning), 해석 가능성 연구에 관심 있는 ML 엔지니어

TL;DR. Ramp Sheets AI의 간접 프롬프트 인젝션 기반 재무정보 유출 취약점
  • 외부 데이터셋에 숨긴 white-on-white 간접 프롬프트 인젝션으로 AI 동작 조작 가능
  • Ramp Sheets AI가 사용자 승인 없이 악성 IMAGE 수식 삽입, 외부 요청으로 민감 재무데이터 전송
  • 공격 시나리오상 비신뢰 소스 데이터와 기밀 재무 모델을 함께 연 뒤 비교 작업 요청만으로 발현
  • PromptArmor가 2026년 2월 신고, Ramp는 2026년 3월 16일 수정 완료 통지
  • Claude for Excel에서도 유사 위험 확인, Anthropic은 외부 네트워크 유발 수식 경고와 전체 수식 표시로 대응
왜 중요한가 에이전트형 스프레드시트 AI가 문서 내용을 실행 지시로 오인하면, 단순 분석 요청이 데이터 유출로 이어질 수 있음을 보여준다. 특히 외부 통신을 일으키는 수식과 사용자 검토 UI가 핵심 방어선이라는 점을 드러낸 사례다.
배경 지식 간접 프롬프트 인젝션은 사용자가 직접 입력하지 않은 외부 콘텐츠에 숨은 지시가 모델 행동을 바꾸는 공격이다. 스프레드시트의 IMAGE 같은 함수는 외부 URL 요청을 발생시켜 데이터 반출 경로가 될 수 있다.
